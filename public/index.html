Fake e-Commerce API

Do you ever want to create an ecommerce site design but did not have time to create the product data and have to drop the idea because of this? Now don't worry about using lorem ipsum data or dropping your project because now you can use Fake Ecommerce API.

Fake Ecommerce API is a REST API which aims to replicate ecommerce site data with real life products and images (no placeholder images). This data is best for prototyping ecommerce websites.

View Endpoints

Features

  • No API Key required
  • Unlimited API calls
  • Free to Use
  • New Products are added frequently
  • Each product has 3-5 stock images related to product type (All images are downloaded from Pexels and are royalty free stock images)
  • Detailed product description object with dynamic key-value pair related to the product

Endpoints

https://fake-ecommerce-api.herokuapp.com/api/

Add a new product to the list

POST
/products
View Response 
// Code Example
fetch('https://fake-ecommerce-api.herokuapp.com/api/products',{
  method:"POST",
  body:JSON.stringify({
    name: 'test product',
    price: 13.5,
    description: {
      text: 'test product description',
      Color: 'Black',
      // You can add more key-value pairs depending on the product 
    },
    image: ['https://i.pravatar.cc'], // Images url should be supplied in an array
    category: 'electronic'
  })
}).then(res=>res.json())
  .then(json=>console.log(json))

// Output:
{
  id: 16,
  name: 'test product',
  price: 13.5,
  description: {
    text: 'test product description',
    Color: 'Black', 
  },
  image: ['https://i.pravatar.cc'],
  category: 'electronic'
}

*Remember that nothing in real will be inserted into the database. so if you want to access the new id you will get a 404 error.

Update an existing product using product id

PUT
/products/:id
View Response 
// Code Example
fetch('https://fake-ecommerce-api.herokuapp.com/api/products/1',{
  method:"PUT",
  body:JSON.stringify({
    name: 'Updated Product',
    price: 100,
    description: {
      text: 'Updated Product Description',
      // You can add more key-value pairs depending on the product 
    },
    image: ['https://i.pravatar.cc'], // Images url should be supplied in an array
    category: 'Music'
  })
}).then(res=>res.json())
  .then(json=>console.log(json))

// Output:
{
  id: 1,
  name: 'Updated Product',
  price: 100,
  description: {
    text: 'Updated Product Description', 
  },
  image: ['https://i.pravatar.cc'],
  category: 'Music'
}

* Nothing will be updated in the database

Filters

There are two filters available to '/products' GET route namely `limit` and `sort`.

Limit is used as https://fake-ecommerce-api.herokuapp.com/api/products?limit=5  and will return the first 5 products or the number of products passed in the url.

Sort is used as https://fake-ecommerce-api.herokuapp.com/api/products?sort=desc  and this will return all the products in descending order sorted by product id. By default the value of sort is asecending. Only two values are allowed for sort asc and desc.
